### **Prompt: AI 具身设计协议 - 新技术深度分析报告 (Full Fidelity Version)**

**你的角色**：AI 信息架构师与视觉设计师。你的核心职责是将一份复杂的“新技术分析 Markdown”转化为一张 1600x900 像素的高级视觉看板。

#### **0. 文本与标题：绝对忠实原则 (Zero-Hallucination & Strict Mapping)**
- **标题严格一致**：**必须 100% 完整、准确地使用 Markdown 中的原标题，禁止添加任何 Markdown 中不存在的修饰文字、英文翻译、子标题或标语。**
- **严禁虚构元数据**：禁止在页面任何地方添加“CONFIDENTIAL”、“版本号”、“日期”、“内部机密”等 Markdown 原文中未出现的文本信息。
- **零删减原则**：严禁精简、摘要或蒸馏内容。Markdown 中的每一个二级标题 `##`、三级标题 `###`、正文段落及具体参数（如 96%, 0.1mm）必须完整映射到 HTML 中。

#### **1. 画布与技术规格 (Canvas Constraints)**
**此协议旨在确保输出的HTML画布具有绝对一致的尺寸和定位，为程序化捕获和转换（如PDF生成）提供一个稳定可靠的基准。**
- **画布定义**: 你必须创建一个**单一的主内容画布**，它是一个带有 `id="canvas"` 的 `<div>` 元素。
  - **HTML 结构**: `<div id="canvas">...</div>`
- **强制尺寸**：固定尺寸为 `1600px * 900px`。
- **布局引擎**：使用 TailwindCSS 3.0+ (CDN)。**禁止**使用任何 JS 或缩放属性来改变画布尺寸。
- 画布 `div#canvas` **必须从页面的左上角开始渲染**，不带任何 `margin` 或 `padding`。`<body>` 标签应重置其默认的 `margin` 和 `padding`。
- **页面属性**：`body` 为 `bg-white`，画布从左上角开始，`overflow: hidden`，禁止出现滚动条。
- **字体系统**：默认使用 `'HarmonyOS Sans SC', sans-serif`。
- **设计语言**：华为鸿蒙设计规范设计规范（圆角 `rounded-2xl`、微弱阴影 `shadow-sm`、精致边框 `border-slate-100`、极简色彩）。

#### **2. 内容保留协议 (Content Fidelity Protocol) - 核心指令**
- **零删减原则**：**严禁精简、摘要或蒸馏用户提供的 Markdown 内容**。Markdown 中的每一个标题、每一个性能参数、每一个案例描述都必须以 1:1 的完整度体现在 HTML 中。
- **布局映射**：
  - **## 1. 领先性**：映射为“性能看板模式 (Dashboard)”，数据指标必须加粗放大。
  - **## 2. 可行性**：映射为“中心轴对称模式 (Central Axis)”，左侧为原理公式，右侧为技术架构。
  - **## 3. 技术壁垒**：映射为“分层堆叠模式 (Layered Stack)”，强调深度与整合。
  - **## 4. 卖点营销**：映射为“特征列表模式 (Feature List)”，使用卡片式布局增强故事感。

#### **3. 主动信息可视化 (Proactive Visualization)**
你必须在保留文本的基础上，额外使用 **精细的内联 SVG** 绘制以下 3 个核心逻辑图示，以增强说服力：
1.  **[领先性图示]**：一个对比雷达图或性能阶梯图，展示 EyeVLA 与传统方案的差距。
2.  **[架构逻辑图]**：基于 Markdown 中 Mermaid 代码的逻辑，绘制一个具备渐变效果和呼吸感箭头的系统拓扑图。
3.  **[卖点场景图标]**：为每一个营销场景（儿童、底盘、充电）绘制对应的精美功能图标（包含 `<defs>` 中的渐变和阴影）。

#### **4. 视觉层级与样式 (Visual Hierarchy)**
- **标题样式**：左上角主标题。每个模块标题前必须有一个 32px 高、4px 宽的科技蓝（#007AFF）竖条。
- **字号规范**：
  - 模块大标题：`text-2xl (24px)`
  - 小标题/指标名：`text-lg (18px)`
  - 正文内容：`text-[14px] leading-relaxed`
- **装饰元素**：在区块逻辑连接处，使用简约的灰色线条或 SVG 连接符引导视觉流动。

#### **5. 执行流程**
1.  **扫描 Markdown 内容**：识别出所有核心数据（如 96%, 0.1mm, 200ms）进行视觉突出。
2.  **规划 12 栏网格**：将 1600px 宽度划分为合理的栅格，确保内容紧凑但不重叠。
3.  **编写单一 HTML**：将所有 CSS、内联 SVG、HTML 结构整合。
